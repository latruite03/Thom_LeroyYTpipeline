# Vidéo 01 — Script (v2 : pédagogie + parcours)

## Titre (provisoire)
**Construire une mini‑app à 40+ (sans être dev) : les outils que j’utilise, étape par étape**

## Intro (texte exact — ~25–40s)
Salut, moi c’est Thom.

J’ai 40+, je vis à Bruxelles, et j’essaie de suivre la tech et l’IA… mais à un rythme adulte.
Sans posture, sans hype.

Dans cette série, le but n’est pas de te “présenter une app”.
Le but, c’est de te montrer le parcours : comment on peut utiliser des outils actuels pour construire quelque chose d’utile, même si on n’est pas développeur de métier.

Aujourd’hui, on part de zéro : qu’est‑ce qu’une app (au fond), et quels outils j’utilise pour en fabriquer une version simple.

## Corps (bullets — pédagogie d’abord)

### 1) C’est quoi “une app” ? (version simple)
- Une app, c’est souvent :
  - une page web (HTML/CSS)
  - un peu de logique (JavaScript)
  - une petite base de données (parfois)
- Et surtout : un problème concret → une interface qui réduit la friction.

### 2) Le problème qu’on choisit (exemple)
- Exemple “Bruxelles Solo” : quand tu es seul, ce n’est pas l’idée qui manque… c’est la décision.
- Donc l’objectif est minimal : proposer 3 idées en fonction de 4–5 critères.

### 3) Les outils qu’on utilise (et à quoi ils servent)
- **Codex / assistant de code** : m’aider à écrire du code propre et rapide (mais je garde la décision).
- **GitHub** : garder l’historique, revenir en arrière, publier.
- **Supabase** : une base de données simple pour stocker les lieux.
- (Option) **Plesk / hébergement** : mettre en ligne.

### 4) Démo “parcours” (pas démo produit)
- Montrer le repo GitHub (un dossier = un outil)
- Montrer la base Supabase (table `places`)
- Montrer la home (les critères → 3 suggestions)

### 5) Ce que tu dois retenir
- Tu n’as pas besoin d’être “dev pur” pour apprendre.
- Tu as besoin d’un problème simple + d’un cadre + d’un petit système.

## Outro (texte exact — ~20–35s)
Si tu veux, je continue la série avec le même principe : un pas concret à la fois.

Et si tu es à Bruxelles : donne‑moi une adresse solo‑friendly (nom + quartier + pourquoi c’est OK). Je l’ajoute à la base.

Sinon : dis‑moi ton problème du quotidien en 2 lignes. On essaie d’en faire un mini‑outil, ensemble.
